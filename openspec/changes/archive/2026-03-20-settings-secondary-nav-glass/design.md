## Context

当前设置页在内容区顶部展示“设置中心”横条，同时左侧列表与右侧内容区处于同一面板内。该布局占用纵向空间、限制内容区可用宽度，并与整体中性扁平风格不够一致。现有实现集中在 `SettingsPanel.vue` 与全局样式 `app.css`，不涉及数据模型或服务端。

## Goals / Non-Goals

**Goals:**
- 移除设置页头部横条与状态展示，让左侧二级菜单成为主导航。
- 将左侧菜单视觉提升为轻玻璃效果，主内容卡片保持实底扁平。
- 扩展设置页可用宽度，提升内容区信息承载能力。

**Non-Goals:**
- 不调整设置页功能项的业务逻辑与交互行为。
- 不引入新的依赖或设计系统变更。
- 不修改主侧边栏（全局导航）结构。

## Decisions

- **保留现有结构，只移除头部横条**：在 `SettingsPanel.vue` 中删除头部区块，减少 DOM 改动并避免影响安装/卸载等流程。
  - 备选方案：新增独立标题栏或折叠标题。未采用，因为目标是释放空间。
- **仅左侧菜单采用玻璃效果**：菜单容器使用半透明背景 + blur，卡片保持实底，确保可读性与信息密度。
  - 备选方案：全页玻璃化。未采用，因为会降低表单与文本对比度。
- **布局宽度扩展到更大 max-width**：`panel-stack` 放宽到约 1200px，左侧菜单固定 180px，内容区自适应剩余宽度。
  - 备选方案：完全拉满宽度。未采用，避免超宽屏阅读分散。

## Risks / Trade-offs

- [玻璃效果在低端设备可能有性能开销] → 只应用于左侧菜单，并保留不支持 `backdrop-filter` 时的实底回退。
- [弱化阴影可能降低层级识别] → 通过边框与留白保持结构清晰。
- [小屏布局挤压] → 保持现有 900px 以下响应式布局（菜单变网格）。

## Migration Plan

- 仅前端样式与模板调整，无数据迁移与兼容性风险。
- 若需要回滚，只需恢复 `SettingsPanel.vue` 头部区块与相关样式。

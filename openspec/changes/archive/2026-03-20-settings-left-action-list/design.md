## Context

当前设置页主要以内容区为核心，缺少内部导航。此次改动在设置页内容区内部左侧加入一列功能选取，作为本地安装、配置与卸载相关操作的入口，视觉与交互保持与现有设置页一致。

## Goals / Non-Goals

**Goals:**
- 在设置页内部左侧新增一列可点击的功能选取列表
- 提供明确的选中态与未选中态样式
- 选中项与右侧内容区展示保持联动

**Non-Goals:**
- 不改变现有功能的业务逻辑与后端接口
- 不重构现有设置页其他模块的布局

## Decisions

- 采用「设置页内部双列布局」：左侧为功能选取列表，右侧为现有内容区。这样保持页面信息层级清晰且可扩展。
- 将功能选取项定义为可复用的列表数据结构（标题、id、状态），便于后续增减条目。
- 选中态通过明显的字体/颜色与背景强调，并与当前内容区模块的可视区域进行同步。

## Risks / Trade-offs

- 左侧新增列可能压缩内容区宽度 → 通过固定列宽与内容区自适应保证主要内容可读性
- 未来功能条目增多可能导致列表过长 → 预留可滚动容器样式

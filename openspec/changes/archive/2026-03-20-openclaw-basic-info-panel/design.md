## Context

当前设置页的“基础配置”默认展示安装向导，缺少 OpenClaw 版本/安装/运行状态的统一入口；且左侧菜单仍包含“关闭重启”占位条目。渲染进程已具备安装、启动 Gateway、读取配置等能力，但缺少对 Gateway 运行状态与停止/重启的统一管理。

## Goals / Non-Goals

**Goals:**
- 在“基础配置”默认展示 OpenClaw 基础信息与操作按钮，安装向导改为隐藏子页。
- 左侧菜单移除“关闭重启”，保持简洁。
- 新增 Gateway 运行状态查询与“启动/关闭/重启”操作，仅控制本应用启动的进程。
- 保持现有 UI 风格，新增轻量信息布局样式。

**Non-Goals:**
- 不检测或管理系统中非本应用启动的 OpenClaw 进程。
- 不新增复杂的状态持久化或日志视图。
- 不修改安装流程的核心逻辑与依赖安装策略。

## Decisions

- **引入隐藏子页 `install-wizard`**：安装向导不再默认显示，点击“安装”按钮切换到隐藏子页；避免页面拥挤，同时保留现有安装流程与组件复用。
  - 备选方案：折叠/展开同页向导。放弃原因：交互状态复杂、布局变化大。

- **Gateway 状态由主进程单一来源维护**：主进程持有 gateway 进程引用，并提供 `gateway:status/stop/restart` IPC。
  - 备选方案：渲染进程通过日志或 URL 推断状态。放弃原因：不可靠且难以处理进程退出。

- **新增 `openclaw:info` 统一基础信息获取**：主进程负责检测 `openclaw -v` 与 gateway 运行状态，渲染进程只负责展示与触发。
  - 备选方案：渲染进程直接调用 `openclaw -v`。放弃原因：执行权限与跨平台一致性差。

## Risks / Trade-offs

- **[Risk] Gateway 进程意外退出导致状态滞后** → Mitigation：监听 `exit` 更新状态并通过事件/轮询刷新。
- **[Risk] stop/restart 在未运行时被触发** → Mitigation：主进程做幂等处理，未运行则直接返回成功或转为 start。
- **[Trade-off] 仅管理本应用启动的进程** → Mitigation：在 UI 中明确说明状态范围，避免误解。

# OpenClaw 安装器 UI 重设计方案（现代扁平 + 玻璃卡片）

## 背景
当前安装器已具备三步流程（安装 / Model API 授权 / 启动控制台），但界面风格偏旧与信息密度偏高。需要整体升级为现代扁平风格，并采用玻璃卡片式设计，保持流程简单。

## 目标
- 视觉上：中性极简 + 冷灰蓝主色，玻璃卡片带厚重层次。
- 体验上：用户只关注一个主区域，操作路径短且直观。
- 结构上：保留分步页面（安装 / 授权 / 启动）并统一视觉框架。
- 反馈上：日志默认可见但不喧宾夺主，错误提示简洁清晰。

## 非目标
- 不改变业务流程或主进程逻辑。
- 不新增新的功能步骤或复杂设置项。
- 不引入深色模式或多主题切换（后续可扩展）。

## 视觉语言
- **风格**：现代扁平 + 玻璃卡片，极简克制。
- **背景**：雾面浅灰→冷白渐变，带柔和光晕。
- **主色**：冷灰蓝（强调按钮、关键状态）。
- **卡片质感**：厚重玻璃，明显分层与阴影。
- **排版**：无衬线现代字体，标题中等字重，正文清爽。
- **装饰**：仅使用细边框与轻线条分区，避免多余图形。

## 页面结构（分步页面）
三页结构一致，仅替换内容模块与文案：

1. **安装页**  
   - 顶部品牌区（Logo/名称/子标题）  
   - 状态胶囊（当前步骤）  
   - 主按钮（开始安装 / 安装中）  
   - 日志摘要区（默认显示）  
  - 错误提示区（出现时替换主操作，主动作“重试”，可附次要“复制日志”）

2. **授权页**  
   - 同样的品牌与状态结构  
   - 主按钮（开始授权 / 授权中）  
   - WebView 内容卡（嵌入授权页面）  
   - 授权码高对比显示  
   - 日志摘要 + 错误提示

3. **启动页**  
   - 同样的品牌与状态结构  
   - 主按钮（启动控制台）  
   - 错误提示（若启动失败）

## 布局与信息层级（方案 A：单主卡）
- 页面中心单一大卡片，承载所有信息。
- 不再使用长步骤列表，改为“当前阶段 + 简短提示”。
- 日志区默认显示，但以低对比度呈现，避免抢占主按钮。

## 交互与状态
- **按钮状态**：运行中显示“进行中…”，禁用并淡化。
- **日志区**：默认显示最近 6 行，超出滚动，不闪烁。
- **错误**：错误提示区出现时替换主操作区域（主动作“重试”，可附次要“复制日志”），日志区仍保留以便排障；错误详情可用次要文本展示。
- **成功**：保持现有流程行为（授权完成进入“启动控制台”页）；仅调整提示文案与视觉，不改变流程逻辑。
- **WebView**：保持可视但不压过主按钮，边界清晰。

## 视觉令牌（建议值）
- **Glass 卡片**  
  - `background: rgba(255,255,255,0.65)`  
  - `backdrop-filter: blur(18px)`  
  - `border: 1px solid rgba(255,255,255,0.35)`  
  - `box-shadow: 0 24px 60px rgba(30,40,60,0.18)`
- **主色（冷灰蓝）**  
  - `#607DFF`（主按钮/高亮）  
  - `#3E4FA8`（深色文本）  
- **中性色**  
  - `#F6F7FA`（背景浅灰）  
  - `#E4E6EE`（边框）  
  - `#6B7280`（辅助文字）  
- **错误色**  
  - `#C54B4B`（低饱和红）

## 排版基准（建议值）
- **字体家族**  
  - 标题：`"Space Grotesk", "Manrope", "Segoe UI", sans-serif`  
  - 正文：`"Manrope", "Inter", "Segoe UI", sans-serif`
- **字号/行高**  
  - 标题（页面主标题）：`24–28px` / `1.2`  
  - 小标题（状态/分区）：`14–16px` / `1.3`  
  - 正文（说明/日志）：`13–14px` / `1.5`  
  - 按钮文字：`14–15px` / `1`

## 动效
- **进入动效**：卡片轻微上浮 + 渐显（300–500ms）。
- **状态切换**：按钮文案淡入淡出。
- **日志更新**：平滑插入，无闪烁。

## 可访问性
- 主按钮与背景对比度 ≥ 4.5。
- 错误提示文字与背景对比度 ≥ 4.5。
- 重要操作不依赖颜色传达，包含文案提示。

## 影响范围
- **前端样式**：`src/renderer/src/styles/app.css`
- **组件结构**：`src/renderer/src/components/*.vue`（保持三页结构，视觉重排）
- **不涉及** 主进程与流程逻辑修改

## 适配与尺寸
- **窗口尺寸**：遵循现有窗口大小（默认约 1100×760），布局自适应缩放。
- **最小可用宽度**：约 760px；小于该宽度时主卡收窄并上下堆叠按钮。
- **关键间距（建议值）**  
  - 主卡宽度：`min(960px, 92vw)`  
  - 主卡内边距：`28–36px`  
  - 模块间距：`16–20px`  
  - 按钮高度：`44–48px`  
  - 日志区高度：`min-height: 96px`

## 文案规范（错误态示例）
- **安装页**：`安装失败，请检查网络或权限后重试。`  
- **授权页**：`授权失败，请重试或检查网络。`  
- **启动页**：`启动失败，请稍后重试。`

## 测试与验收
- 启动 `npm run dev`，确认三步页面正常显示。
- 检查按钮、日志区、错误提示在三页均可正常渲染。
- 确认 WebView 卡片在授权页内嵌显示、布局不溢出。
